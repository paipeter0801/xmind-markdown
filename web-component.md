* **1. 導覽與選單 (Navigation & Menus)**
* **Breadcrumb (麵包屑)**
* 外觀：由大於符號或斜線分隔的文字路徑 (例：首頁 > 分類 > 產品)。
* 功能：顯示目前在網站層級中的位置，並允許快速返回上一層。
* 適用場景：層級較深的電商網站、文件系統導覽。


* **Context Menu (右鍵選單)**
* 外觀：隱藏的下拉列表，右鍵點擊特定區域後浮現。
* 功能：提供與當前右鍵點擊對象相關的進階操作。
* 適用場景：檔案管理系統、對特定表格資料列進行編輯或刪除。


* **Dropdown Menu (下拉選單)**
* 外觀：點擊按鈕或圖示後，向下展開的操作列表。
* 功能：收納多個選項或操作，節省畫面空間。
* 適用場景：右上角的使用者帳號選單、操作設定按鈕 (點擊出現：編輯/複製/刪除)。


* **Menubar (選單列)**
* 外觀：水平排列的常駐選單列（類似桌面軟體頂部）。
* 功能：組織與收納複雜的應用程式全局功能。
* 適用場景：雲端文件編輯器 (檔案/編輯/檢視)、複雜的 Web 應用系統。


* **Navigation Menu (導覽選單)**
* 外觀：頂部的水平導覽列，常伴隨滑鼠懸停展開的大型面板（Mega Menu）。
* 功能：網站的主要路由導覽。
* 適用場景：企業官網主導覽列、SaaS 產品介紹頁頂部。


* **Pagination (分頁控制)**
* 外觀：一排數字按鈕，搭配「上一頁/下一頁」圖示。
* 功能：將大量資料切分為多頁顯示，方便導覽。
* 適用場景：商品列表頁、部落格文章列表、數據報表。


* **Sidebar (側邊欄)**
* 外觀：固定在畫面左側或右側的垂直區塊。
* 功能：提供應用程式的全局主要導覽或附屬資訊。
* 適用場景：後台管理系統的主要選單、儀表板介面。


* **Tabs (頁籤 / 分頁)**
* 外觀：水平或垂直排列的標籤列。
* 功能：在同一個畫面的同一區塊內，切換顯示不同的內容視圖。
* 適用場景：設定頁面中的分類（如：一般設定 / 隱私 / 外觀）、商品詳情與評價切換。




* **2. 資料輸入與表單 (Inputs & Forms)**
* **Button (按鈕)**
* 外觀：矩形或圓角的可點擊區域，包含文字或圖示。
* 功能：觸發特定動作或提交表單。
* 適用場景：確認、取消、送出表單、任何需要使用者點擊觸發的行為。


* **Button Group (按鈕群組)**
* 外觀：多個按鈕緊密相連成一排，邊界相接。
* 功能：將功能相關的操作按鈕分組在一起。
* 適用場景：文字對齊選擇（左/中/右）、切換不同社群分享。


* **Checkbox (核取方塊)**
* 外觀：小方格，點擊後會出現打勾標記。
* 功能：允許使用者選取一個或多個選項。
* 適用場景：同意服務條款、列表的多重篩選條件。


* **Combobox (組合方塊)**
* 外觀：結合「文字輸入框」與「下拉選單」的複合元件。
* 功能：可輸入文字快速搜尋，並從過濾後的清單中選擇。
* 適用場景：國家/城市列表選擇、大量選項（如幾百個 Tag）需快速過濾時。


* **Date Picker (日期選擇器)**
* 外觀：點擊輸入框會彈出日曆面板。
* 功能：讓使用者透過視覺化介面選擇特定日期。
* 適用場景：生日填寫、預約日期、出發時間設定。


* **Range Calendar (日期範圍日曆)**
* 外觀：日曆面板，允許選取「開始」與「結束」兩個日期。
* 功能：選擇一段連續的時間範圍。
* 適用場景：飯店預訂區間、報表資料搜尋的時間區間。


* **Input (單行輸入框)**
* 外觀：基礎的單行文字欄位。
* 功能：接收使用者輸入的簡短文字。
* 適用場景：帳號、密碼、姓名填寫、搜尋框。


* **Input Group (輸入框群組)**
* 外觀：輸入框兩側附帶圖示、按鈕或固定文字後綴。
* 功能：提供輸入框額外的上下文或附屬操作。
* 適用場景：金額輸入(前置 $ 符號)、網址輸入(前置 https://)、帶有搜尋按鈕的搜尋框。


* **Input OTP (一次性密碼輸入)**
* 外觀：多個獨立的單字元輸入小方塊。
* 功能：專門用於輸入驗證碼，輸入一格會自動跳下一格。
* 適用場景：雙重驗證 (2FA)、手機簡訊驗證碼。


* **Label (標籤)**
* 外觀：伴隨輸入框的說明文字。
* 功能：說明輸入框的用途，點擊文字可自動聚焦對應的輸入框。
* 適用場景：所有表單欄位的標題。


* **Native Select (原生選擇器)**
* 外觀：呼叫瀏覽器或手機作業系統預設下拉選單。
* 功能：最基礎的單選下拉選單。
* 適用場景：重視行動端原生體驗、功能簡單的選項表單。


* **Select (自訂選擇器)**
* 外觀：樣式可完全自訂的下拉選單。
* 功能：從預設列表中進行單選。
* 適用場景：需要整體網站 UI 風格高度一致的下拉選單。


* **Radio Group (單選按鈕群組)**
* 外觀：一組小圓圈，選中時內部會出現實心圓點。
* 功能：限制使用者在一組互斥選項中，只能選擇其中一個。
* 適用場景：性別選擇、付款方式選擇。


* **Slider (滑桿)**
* 外觀：一條軌道加上可左右拖曳的滑塊。
* 功能：在連續的數值區間內快速調整數值。
* 適用場景：音量控制、影片進度條、價格區間拉桿。


* **Switch (開關)**
* 外觀：橢圓形開關，點擊時滑塊會左右切換並變色。
* 功能：控制特定功能的開啟或關閉狀態。
* 適用場景：深色/淺色模式切換、接收推播通知開關。


* **Textarea (多行輸入區)**
* 外觀：較大的多行文字輸入框，通常右下角可拖曳放大。
* 功能：接收長篇幅的文字輸入。
* 適用場景：聯絡我們表單、文章編輯、自我介紹。


* **Toggle (狀態切換按鈕)**
* 外觀：類似一般按鈕，但點擊後會保持在「被按下」的視覺狀態。
* 功能：切換單一選項的開啟/關閉狀態。
* 適用場景：文字編輯器的「加粗(B)」、「斜體(I)」、收藏愛心按鈕。


* **Toggle Group (狀態切換群組)**
* 外觀：一組 Toggle 按鈕的集合。
* 功能：在一組狀態中進行單選或多選。
* 適用場景：文字編輯器的對齊方式（左/中/右只能單選）。


* **Field / Formsnap (表單結構整合)**
* 外觀：無特定外觀，屬於代碼層級的封裝。
* 功能：整合標籤、輸入框、錯誤訊息，並處理表單驗證狀態。
* 適用場景：開發者在建構複雜、需要即時驗證錯誤的表單時使用。




* **3. 彈出視窗與覆蓋層 (Overlays & Dialogs)**
* **Dialog (對話框 / 模態視窗)**
* 外觀：螢幕中央彈出的懸浮視窗，背景通常會變暗。
* 功能：在不離開當前頁面的情況下，強制使用者關注並處理特定內容或表單。
* 適用場景：新增使用者表單、查看資料詳細資訊、結帳視窗。


* **Alert Dialog (警告對話框)**
* 外觀：類似 Dialog，但通常包含強烈的警告色調或提示。
* 功能：中斷操作流程，要求使用者進行具風險或破壞性操作的確認。
* 適用場景：確認刪除帳號、未儲存離開頁面的警告。


* **Drawer (抽屜面板)**
* 外觀：從螢幕邊緣（最常見為下方）滑出並覆蓋部分畫面的面板。
* 功能：提供類似 Dialog 的功能，但更適合行動裝置單手操作的操作體驗。
* 適用場景：行動端網頁的底部選單、社群軟體的分享選項面板。


* **Sheet (側滑面板)**
* 外觀：從螢幕側邊（通常為右側）滑出的大型垂直面板。
* 功能：處理需要較多空間的複雜表單或設定，且不完全遮擋畫面。
* 適用場景：電商網站的側邊購物車預覽、進階篩選器面板。


* **Popover (彈出框)**
* 外觀：點擊某個元素後，在該元素附近就地彈出的浮動小面板。
* 功能：顯示進階設定、小型表單或額外資訊，點擊面板外即可關閉。
* 適用場景：複雜的日期選擇器面板、篩選條件的細部設定。


* **Hover Card (懸浮卡片)**
* 外觀：滑鼠懸停 (Hover) 於文字或圖示上時，短暫延遲後彈出的小浮窗。
* 功能：提供該連結或元素的進階預覽資訊，無須點擊跳轉。
* 適用場景：游標停留在使用者名稱時，彈出其大頭貼與個人簡介預覽。


* **Tooltip (工具提示)**
* 外觀：滑鼠懸停時立即出現的黑色小文字方塊。
* 功能：僅用於解釋按鈕或圖示的簡單作用。
* 適用場景：純圖示按鈕（如垃圾桶圖示旁提示「刪除」）、介面名詞解釋。




* **4. 資料展示 (Data Display)**
* **Accordion (手風琴 / 折疊面板)**
* 外觀：垂直排列的標題列表，點擊標題可展開/收合對應的內容區域。
* 功能：在有限的空間內分層隱藏/顯示大量分區資訊，減少視覺負擔。
* 適用場景：常見問題 (FAQ) 頁面、設定選單中的細項分類。


* **Avatar (頭像)**
* 外觀：圓形或方形的圖片區域。
* 功能：代表使用者或組織的視覺標識，若無圖片可自動轉為名稱縮寫 (如 JD)。
* 適用場景：使用者大頭貼、評論區留言者頭像。


* **Badge (徽章 / 標記)**
* 外觀：帶有背景色與圓角的小型文字區塊。
* 功能：用來標記狀態、屬性或分類。
* 適用場景：訂單狀態（已出貨/處理中）、文章分類標籤、未讀訊息數字。


* **Card (卡片)**
* 外觀：帶有邊框與些微陰影的矩形容器，通常包含標題、內容與底部按鈕。
* 功能：將相關的一組資訊與操作打包成一個獨立的視覺單元。
* 適用場景：電商商品卡片、部落格文章預覽區塊、後台儀表板的數據方塊。


* **Carousel (輪播圖)**
* 外觀：可左右滑動切換的內容列表。
* 功能：在固定空間內輪流展示多張圖片或多張卡片。
* 適用場景：首頁主視覺 Banner、商品詳情頁的多角度圖集。


* **Chart (圖表)**
* 外觀：長條圖、折線圖、圓餅圖、雷達圖等。
* 功能：將抽象的數據可視化，幫助使用者快速理解趨勢與比例。
* 適用場景：後台數據分析儀表板、財務報表走勢。


* **Data Table (資料表格)**
* 外觀：帶有分頁、列首排序、全域搜尋、條件篩選等進階功能的表格。
* 功能：用於管理與高效率操作大量的結構化資料。
* 適用場景：後台的使用者名單管理、訂單管理系統。


* **Table (基礎表格)**
* 外觀：基礎的網格表格，無複雜的互動功能。
* 功能：單純、靜態地展示行列對應的資料。
* 適用場景：發票明細展示、產品規格比較表。


* **Empty (空狀態)**
* 外觀：通常包含一個插圖、一句提示文字與一個行動呼籲 (CTA) 按鈕的置中區塊。
* 功能：在系統沒有資料可顯示時，給予使用者明確回饋並引導下一步。
* 適用場景：購物車為空時提示「去逛逛」、搜尋無結果時提示「換個關鍵字」。


* **Kbd (鍵盤快捷鍵標示)**
* 外觀：模仿實體鍵盤按鍵樣式的小型文字框。
* 功能：明確提示使用者該功能支援的鍵盤快捷鍵。
* 適用場景：搜尋框旁提示 `Ctrl` + `K`、選單項目的快捷鍵說明。


* **Calendar (日曆)**
* 外觀：標準的月份日期網格。
* 功能：單純展示日期，或作為 Date Picker 的基礎底層元件。
* 適用場景：行事曆應用、打卡紀錄展示。


* **Typography (排版樣式)**
* 外觀：各種標準化大小、粗細、行高的文字設定。
* 功能：統一整個網站的字體視覺風格（包含大標題、內文、引言、代碼區塊）。
* 適用場景：文章閱讀頁面、全局 CSS 字體設定。


* **Item (子項目)**
* 外觀：無獨立外觀，依附於父元件。
* 功能：作為列表或選單（如 Dropdown, Select）中的單一選項元素。
* 適用場景：建構選單內容時的基礎單位。




* **5. 回饋與狀態 (Feedback & Status)**
* **Alert (提示訊息框)**
* 外觀：帶有特定顏色邊框（紅/綠/藍）與對應圖示的訊息方塊。
* 功能：在頁面佈局中靜態顯示重要提示、成功訊息或警告，不會自動消失。
* 適用場景：系統維護公告、表單提交錯誤時列出錯誤原因。


* **Sonner / Toast (吐司通知)**
* 外觀：在螢幕角落（通常為右下或頂部）短暫滑出，數秒後自動消失的浮動小通知。
* 功能：提供操作結果的輕量級、非干擾性回饋。
* 適用場景：「設定已成功儲存」、「網址已複製到剪貼簿」。


* **Progress (進度條)**
* 外觀：水平的軌道長條，內部填色長度代表百分比。
* 功能：顯示耗時任務的完成進度。
* 適用場景：檔案上傳進度、多步驟表單的完成度。


* **Skeleton (骨架屏)**
* 外觀：呈現內容大致輪廓的灰色閃爍區塊（通常是灰色的圓形或矩形）。
* 功能：作為資料載入時的佔位符，降低使用者等待的焦慮感，比傳統轉圈圈體驗更好。
* 適用場景：網頁初次載入時的文章列表預覽、等待 API 回應的畫面。


* **Spinner (載入動畫)**
* 外觀：旋轉的圓圈、點點或圖示。
* 功能：明確表示系統正在處理中，請稍候。
* 適用場景：按鈕點擊送出後的按鈕內部狀態、小區塊的資料讀取。




* **6. 排版與功能結構 (Layout & Utility)**
* **Aspect Ratio (寬高比容器)**
* 外觀：無形容器，但會強制限制作為其內容的尺寸。
* 功能：強制區塊保持特定的長寬比例 (如 16:9, 1:1)。
* 適用場景：確保嵌入的 YouTube 影片或上傳的圖片在各種螢幕尺寸下皆不變形。


* **Collapsible (可折疊區域)**
* 外觀：包含一個觸發按鈕與一段可隱藏/展開的內容區塊。
* 功能：單純的展開/收合內容（類似單獨一個 Accordion）。
* 適用場景：表單中的「顯示進階設定」、程式碼區塊展開。


* **Command (命令面板)**
* 外觀：帶有搜尋框與分類列表的覆蓋面板，常透過快捷鍵喚醒。
* 功能：允許使用者透過鍵盤快速搜尋網站內容、跳轉頁面或直接執行系統命令。
* 適用場景：類似 Mac 的 Spotlight 搜尋、開發者工具的全局搜尋框。


* **Resizable (可調整大小面板)**
* 外觀：在兩個相鄰面板之間，帶有一條可拖曳的把手（Handle）。
* 功能：允許使用者自行調整左右或上下相鄰兩個面板的寬度或高度。
* 適用場景：類似 VSCode 或 Figma 等專業軟體的側邊欄寬度調整。


* **Scroll Area (滾動區域)**
* 外觀：具有跨平台與跨瀏覽器一致外觀的自訂捲軸區塊。
* 功能：取代系統原生的醜陋捲軸，並且只在滑鼠懸停或滾動時顯示捲軸，節省空間。
* 適用場景：對話紀錄的長區塊、長篇條款閱讀視窗、側邊選單的滾動。


* **Separator (分隔線)**
* 外觀：一條極細的橫線或直線。
* 功能：在視覺與結構上分隔不同的內容區塊，增加版面呼吸空間。
* 適用場景：文章段落之間、選單項目分組之間、卡片標題與內容之間。
