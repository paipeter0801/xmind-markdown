這是一個非常實用且有趣的專案！從你上傳的 `26-zerodawn.xmind` 檔案內容來看，這屬於 **XMind 的 XML 結構版本**（主要是 XMind 8 及更早版本使用的格式，雖然新版 XMind 也能向下相容開啟）。

XMind 檔案本質上是一個 **ZIP 壓縮檔**。要將 Markdown 轉換成這種格式，你需要處理資料結構的解析、XML 檔案的生成，最後將它們打包成 ZIP。

以下是開發「Markdown 轉 XMind (XML版)」功能的 **5 個核心轉換重點**：

### 1. 檔案結構與打包 (ZIP 壓縮)

你的程式最終產出的 `.xmind` 檔案，必須是一個包含特定目錄與檔案的 ZIP 壓縮檔。你必須確保產生的 ZIP 內至少包含以下核心檔案：

* **`content.xml`**：最核心的檔案，存放所有心智圖的節點結構與文字內容。
* **`meta.xml`**：存放檔案的建立時間、建立者、XMind 版本等元數據。
* **`META-INF/manifest.xml`**：這是一個清單檔，宣告了壓縮檔內包含的所有檔案清單。**如果漏掉這個檔案或內容不符，XMind 會將檔案判定為損毀。**
* *(可選)* `styles.xml`：存放主題與樣式設定（如果不特別指定，XMind 會套用預設樣式）。

### 2. Markdown 解析與層級映射 (AST 轉換)

你必須先將 Markdown 的文本解析成「樹狀結構 (Tree)」，才能對應到心智圖的節點。

* **標題層級映射**：
* Markdown 的 `#` (H1) 通常作為心智圖的**中心主題 (Central Topic)**。
* `##` (H2) 作為**主要分支 (Main Topics)**。
* `###` (H3) 及以下，作為**子分支 (Subtopics)**。


* **清單層級映射**：如果使用者是用縮排清單 (`-` 或 `*`) 來寫 Markdown，你需要利用縮排深度來判斷父子節點關係。
* **節點屬性處理**：Markdown 中的超連結 `[文字](網址)` 可以轉換為 XMind 節點的超連結屬性 (`xlink:href`)。

### 3. `content.xml` 的節點生成邏輯

這是轉換中最費工的一環。從你提供的檔案中可以觀察到，XMind 的節點是用 `<topic>` 標籤包裹，且具備嚴格的父子嵌套規則。
轉換時必備的屬性與標籤：

* **`id` 屬性**：每個 `<topic>` 都**必須有全局唯一的 ID**（例如 `id="72ghjekcad670347jgslclo4o9"`）。你可以用隨機生成的英數組合或 UUID 去掉橫線來代替。
* **`timestamp` 屬性**：每個節點需要一個 Unix 毫秒時間戳記。
* **`<title>` 標籤**：存放 Markdown 轉換過來的純文字（例如 `<title>LINE BOT 互動界面</title>`）。
* **`<children>` 與 `<topics type="attached">**`：用來包裹子節點。
* **轉換結構範例**：
```xml
<topic id="parent_id" timestamp="1234567890123">
    <title>父節點名稱</title>
    <children>
        <topics type="attached">
            <topic id="child_id" timestamp="1234567890124">
                <title>子節點名稱</title>
            </topic>
        </topics>
    </children>
</topic>

```





### 4. 進階標記與狀態處理 (Markers & Folded)

如果你的 Markdown 支援特殊語法，可以對應到 XMind 的進階功能（在你上傳的檔案中也有出現）：

* **收合狀態 (`branch="folded"`)**：如果你的 Markdown 有標記某些段落預設收合，可以在該 `<topic>` 加上 `branch="folded"` 屬性。
* **圖示標記 (`<marker-refs>`)**：你可以自訂語法（例如在 Markdown 寫 `[red-flag]`），並在轉換時加入標記：
```xml
<marker-refs>
    <marker-ref marker-id="flag-red"/>
</marker-refs>

```



### 5. `manifest.xml` 清單對齊

每當你生成了 `content.xml` 或 `meta.xml`，都必須在 `META-INF/manifest.xml` 中註冊它。它的結構很簡單，但絕對不能漏：

```xml
<manifest xmlns="urn:xmind:xmap:xmlns:manifest:1.0">
    <file-entry full-path="content.xml" media-type="text/xml"/>
    <file-entry full-path="META-INF/manifest.xml" media-type="text/xml"/>
    </manifest>

```

---

**總結開發流程**：
讀取 Markdown -> 解析成 Tree 物件 -> 遞迴遍歷 Tree 生成 `content.xml` -> 生成 `meta.xml` 與 `manifest.xml` -> 將這些字串打包成 ZIP 並將副檔名存為 `.xmind`。
